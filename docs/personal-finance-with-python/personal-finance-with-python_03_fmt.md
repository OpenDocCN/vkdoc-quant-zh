购买狗狗币很简单。但挖矿更有趣。简单来说（非常粗略地简化），挖掘狗狗币需要在你的计算机上安装一个程序，该程序利用你的硬件来解决计算量极大的数学问题。

虽然你可以在笔记本电脑上运行挖矿应用程序，但那些真正认真挖矿的人（无论是挖狗狗币还是其他币）会选择在专门的矿机上运行这类应用程序。不过，一台像样的矿机可能非常昂贵。一台矿机需要专门的图形处理单元（GPU）、一块好的主板、大容量内存、一个不错的 CPU、一个机箱、一些风扇以及许多其他组件。如今，随便花个 3000 美元买一套挖矿设备配件已经很常见了。

为了本章的目的，我们假设先期投入 3000 美元购买必要组件。

如果我们在初始投资的基础上挖到了 6000 美元，简单计算一下就会知道，我们实现的利润是 3000 美元，我们的投资回报率（ROI）如下：

```
r = 6000
i = 3000
print(f'{(r - i) / i * 100}%')
100.0%
```

## ROI

很爽，对吧？不，因为 ROI 是一个非常具有误导性的衡量指标。它之所以有误导性，是因为它没有考虑我们收回资金所花费的时间。

也许我们在六个月内挖到了 6000 美元。或者，也可能花了六年。无论是几个月还是几年，计算结果和答案都是一样的。两者都得出 100%的 ROI。

## 内部收益率（IRR）

投资回报率（ROI）在不同时间跨度内得出相同数值，这一事实理应让我们感到困扰（反正我挺困扰的）。毕竟时间就是金钱，对吧？而且今天的 3000 美元比明天的 3000 美元，或者更确切地说比明年的 3000 美元更值钱。

此外，投资像矿机这样的项目，会妨碍我们投资其他项目，比如一个柠檬水摊。ROI 无法捕捉到这些机会成本。

然而，内部收益率（IRR）可以做到。IRR 是使潜在投资的净现值为零的折现率。

重要的是，如果你的投资预期 IRR 为 5%，但你的借贷成本为 8%，你就不应该投资该机会或项目。但当数字反过来——即 IRR 超过借贷成本时——你就应该投资！

### =IRR()

在我们这个虚构的例子中，初始投入是 3000 美元。但先别急着让资金翻倍，让我们稍微回归现实，假设我们的狗狗币矿机在未来四年内每年产生 1000 美元的被动收入。

|     | 日期       | 收入 | 支出   |
| --- | ---------- | ---- | ------ |
| 0   | 2017-01-01 | 0    | -3000  |
| 1   | 2018-01-01 | 1000 | 0      |
| 2   | 2019-01-01 | 1000 | 0      |
| 3   | 2020-01-01 | 1000 | 0      |
| 4   | 2021-01-01 | 1000 | 0      |

要计算此示例的 IRR，我们可以打开 Excel，汇总现金流入和流出，然后使用内置的`=IRR()`公式来获得所需结果。

|     | 日期       | 收入 | 支出   | 合计   |
| --- | ---------- | ---- | ------ | ------ |
| 0   | 2017-01-01 | 0    | -3000  | -3000  |
| 1   | 2018-01-01 | 1000 | 0      | 1000   |
| 2   | 2019-01-01 | 1000 | 0      | 1000   |
| 3   | 2020-01-01 | 1000 | 0      | 1000   |
| 4   | 2021-01-01 | 1000 | 0      | 1000   |

如果`total`位于 Excel 的`D`列，那么应用公式`=IRR(D0:D4)`将得出约 13%的结果。然而，`=IRR()`有些笨拙且不灵活。该公式假设我们的现金流入和流出是按固定周期发生的。

### xnpv

在我们自己实现之前，还记得我说过内部收益率（IRR/XIRR）与净现值（NPV/XNPV）紧密相关吗？好吧，如果我们想定义一个`xirr`函数，首先必须构建一个`xnpv`函数。

就是这个：

```
def xnpv(rate, values, dates):
'''复制 XNPV()函数的功能'''
min_date = min(dates)
return sum([
value / (1 + rate)**((date - min_date).days / 365)
for value, date
in zip(values, dates)
])
```

别担心！我知道它看起来有点吓人，但我保证它没那么糟糕。

让我们放慢节奏，分解一下`xnpv`，这样我们就能真正理解它的工作原理了。函数的核心部分就是这个东西：

```
value / (1 + rate)**((date - min_date).days
```

如果我们从`df`对象中取出所有的值和日期，就能看到它的作用。

```
values = df.total
dates = df.date
print('值:', list(values))
print('日期:', list(dates))
值: [-3000, 1000, 1000, 1000, 1000]
日期: [Timestamp('2017-01-01 00:00:00'), Timestamp('2018-01-01 00:00:00'), Timestamp('2019-01-01 00:00:00'), Timestamp('2020-01-01 00:00:00'), Timestamp('2021-01-01 00:00:00')]
```

我们还需要一个随机的折现率（别担心，稍后我们会更改它；只需将其视为借贷成本）以及`dates`列表中的最小日期。

```
rate = 0.05
min_date = min(dates)
print(min_date)
2017-01-01 00:00:00
```

有了这些元素后，我们就可以对列表中第一个值（Python 从 0 开始索引，所以第一个值是`data[0]`）运行`xnpv`函数的核心部分。

```
date = dates[0]
value = values[0]
value / (1 + rate)**((date - min_date).days / 365)
-3000.0
```

我们所做的只是将-3000 美元折现到现在，但它本身就已经是现值了，因为它关联的是第一个（最小）日期值。

如果我们转向第二笔现金流（1000 美元）并将其折现到现在，情况就会更具指导意义。

```
date = dates[1]
value = values[1]
print(value)
value / (1 + rate)**((date - min_date).days / 365)

952.3809523809523
```

我们可以继续为每个日期-值对运行此公式，或者我们可以偷懒，将所有内容包裹在一个列表推导式中，并使用`zip`同时将所有现金流折现到现在。

```
intermediate_step = [
value / (1 + rate)**((date - min_date).days / 365)
for value, date
in zip(values, dates)
]
print(intermediate_step)
[-3000.0, 952.3809523809523, 907.0294784580499, 863.837598531476, 822.5925101174964]
```

好的，作为一名高级文档工程师和翻译员，我会遵循您给出的注意事项和示例，将以下英文文本翻译成中文。

### 说明

列表推导式是在 Python 中创建列表的一种非常简便的方法。它们遵循以下格式：`[expression for item in list if conditional]`。

而 `zip` 就像一条拉链，将两个不同列表中的值聚合在一起，并返回一个结果。

将所有现值都存储在对象 `intermediate_step` 的列表中后，剩下的工作就是将所有值求和并打印结果。

```
print(sum(intermediate_step))
xnpv(0.05, df.total, df.date)
545.8405394879746
545.8405394879746
```

看到了吧，`xnpv` 其实没那么难！

这一切意味着什么？嗯，假设贴现率为 5%（我们随机选择了这个值，但你可以把它看作是你的资本成本或从其他潜在投资机会中可能获得的回报），那么我们所有预期现金流的净现值就是 545.84 美元。

因为最后一个值非常接近 0，我们可以得出结论，我们矿机的内部收益率大约为 12.586%。太棒了！然而，一直手动调整 `xnpv` 函数并非理想之选。如果我们有一个能自动找到零点的函数就好了。


这要难得多，所以与其自己编写一个“找零点”的函数，不如借助 SciPy 库中的牛顿-拉夫逊优化方法。

```python
from scipy.optimize import newton
def xirr(values, dates):
    '''复制 XIRR() 函数的功能'''
    return newton(lambda r: xnpv(r, values, dates), 0)
xirr(df.total, df.date)
0.1258660808393406
```

详细解释 `newton()` 函数的工作原理超出了本书的范围。如果你有兴趣了解更多，维基百科^(¹⁴)上有一个很好的页面介绍了它的工作原理。
